## Risks & Mitigation

### Risk 1: Migration Script Breaks Existing Data
**Likelihood:** Low | **Impact:** High
**Mitigation:**
- Mandatory dry-run testing on backup database
- Data Architect creates database snapshot before migration
- CEO reviews migration output before production run

### Risk 2: Research Analyst Creates Duplicates During Ingestion
**Likelihood:** Medium | **Impact:** Medium
**Mitigation:**
- DevOps Engineer's monitoring script will detect duplicates
- Entity resolution protocol explicitly requires Wikidata Q-ID check first
- Technical Writer documents common pitfalls in real-time

### Risk 3: Sprint Scope Too Ambitious
**Likelihood:** Medium | **Impact:** Low
**Mitigation:**
- 50+ figures is stretch goal; 30+ figures is minimum viable
- CREATED_BY implementation is highest priority (Phase 2.1)
- Documentation can extend into next sprint if needed
